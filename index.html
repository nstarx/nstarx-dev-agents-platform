How They Complement Each Other

Pulse Prana handles the strategic layer (what to build & why). Execution Flow handles the tactical layer (how to build & when). Together they cover the full project lifecycle.

For AI Agents

Both services expose machine-readable endpoints designed for AI agent consumption. Feed these URLs to Claude, GPT, or any coding agent to give them full context about your projects.

Pulse Prana — Know Me

GET /pulse/api/know-me

Returns a comprehensive text description of the entire service: all API endpoints, data models, authentication methods, real-time features, and usage patterns for AI agents.

Execution Flow — Health & Projects

GET /execution/api/health

Health check, project listing, and execution plan API. Includes tasks, phases, team scenarios, risk factors, and critical path data.
